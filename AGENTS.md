# Git Commit Message Guidelines

When writing commit messages, please follow these guidelines to ensure clarity and consistency:

1. **Gitmoji**: Use Gitmoji to categorize your commits.
2. **Semantic Commit Message**: Follow the format `<type>(<scope>): <subject>`, where:
   - **type**: The type of change (e.g., feat, fix, docs, style, refactor, test, chore).
   - **scope**: The area of the codebase affected (optional).
   - **subject**: A brief description of the change.
3. Please check previous commit messages for examples of how to format your messages correctly when you are unsure.
